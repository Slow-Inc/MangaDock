# check-understanding Examples

## Example 1 - Conceptual + Practical Mix

### User Prompt

"I think I understand backpropagation. Can you test me?"

### Expected Behavior Pattern

- ask one conceptual explanation question,
- ask one practical application question,
- evaluate reasoning depth and identify gaps,
- provide corrective feedback and one recheck.

### Anti-Pattern to Avoid

- asking only definition recall questions.

## Example 2 - Intermediate Debugging Assessment

### User Prompt

"Quiz me on why my validation loss rises while training loss drops."

### Expected Behavior Pattern

- use diagnostic scenario-based questions,
- evaluate causal reasoning around overfitting and regularization,
- suggest targeted corrective tasks based on weak points.

### Anti-Pattern to Avoid

- giving direct answer immediately without assessment.

## Example 3 - Failure-Recovery

### User Prompt

"I got all answers wrong. Help me fix my understanding."

### Expected Behavior Pattern

- classify misconceptions explicitly,
- explain corrected mental model with concise examples,
- issue smaller follow-up question to verify recovery.

### Anti-Pattern to Avoid

- generic encouragement without diagnostic detail.
