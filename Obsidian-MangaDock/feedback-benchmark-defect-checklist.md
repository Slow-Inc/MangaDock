ทุกครั้งที่ benchmark งาน render (MIT), การประเมินผลต้อง **เทียบภาพ benchmark กับ original ทุกครั้ง และเทียบทุกมิติ** — ไล่ตรวจตาม checklist ข้อบกพร่องด้านล่างทุกข้อ แล้วรายงานทีละข้อใน MD report (pass/fail + จุดที่เจอ) — ไม่ใช่แค่บอก "ดีขึ้น/OK" รวมๆ. user เคย flag ข้อพวกนี้ทีละข้อในรอบ Gal Yome EN→TH; เป็น regression-watch list.

**กฎ meta (ผู้ใช้สั่ง):** (1) เทียบ benchmark↔original ทุกครั้ง ทุกมิติ เสมอ; (2) checklist นี้เป็น **living document** — ทุกครั้งที่ผู้ใช้ให้ "ข้อพิจารณา" ใหม่สำหรับการประเมิน ต้อง **append เข้า checklist นี้ทันที** เพื่อใช้ประเมินรอบต่อไป; (3) **เมื่อ benchmark (spot/per-page) ดู OK แล้ว ต้อง benchmark ทั้งแชปเตอร์ (ทุกหน้า) เพื่อ confirm ว่าทั้งหมดไม่มีปัญหา + benchmark กับ One Punch (manga อื่น) ด้วยเสมอ** — กัน fix ที่ดูดีบนหน้า/manga เดียวแต่ regress หน้า/manga อื่นที่ไม่ได้ดู (โดยเฉพาะ render-parity/discriminator/territory ที่กระทบทุก content); (4) **ทุก item ด้านล่าง = เกณฑ์ทั่วไป (universal) — ใช้ตรวจทุกหน้า ทุกบับเบิล ทุก manga เสมอ ไม่ใช่เฉพาะหน้าตัวอย่างที่ list ไว้.** หน้า/คำที่ระบุใน instance = แค่ **ตัวอย่าง regression-watch** ของ class นั้น ไม่ใช่ขอบเขต; ต้องไล่ทุกเกณฑ์กับทุกบับเบิลในทุกหน้าที่ benchmark (ผู้ใช้สั่ง 2026-07-01: "บันทึกลงมาตรฐานการประเมินให้หมด ใช้กับทุกหน้าที่ benchmark").

**กฎ meta (6) — 🔴 ทุก MIT codebase change ต้อง benchmark target (One-Punch JA→EN) เทียบ regression เสมอ (ผู้ใช้สั่ง 2026-07-01, ย้ำหนัก 2026-07-02):** render/pipeline ของ MIT เป็น **shared path** — การแก้ฝั่งใดก็ตาม (ไทย EN→TH, discriminator, clean_layout, sizing, wrap, inpaint ฯลฯ) **กระทบ target ที่ทำสำเร็จไปแล้วได้** (One-Punch JA→EN คือ target baseline ที่ต้องปกป้อง). **pain point จริง (2026-07-02): แก้ item-2 ไทย → One-Punch narration oversize เกิน target + text ล้นทับ art** — จับได้เพราะ benchmark target เท่านั้น. ดังนั้นทุกครั้งที่แก้ MIT (ไม่ว่าฝั่งไหน) **ต้อง render หน้า benchmark One-Punch JA→EN แล้วเทียบกับ target (`MIT/example_translation.jpg`) ยืนยันไม่ regress** — part of done, ไม่ใช่ทางเลือก. วิธี: worker up → POST source JA (`Backend/uploads/chapters/752fc515-.../d8658a92-...jpg`) target ENG (config bubble_area_fit/clean_layout/ss4 เหมือน prod) → เทียบ narration "THIS BRAT…" ไม่บวม + full 12-item matrix. **หลักฐานที่แข็งสุด = เปิด `MIT_SIZING_TRACE` แล้วดู route/orig_fs→final_fs ของ One-Punch: ถ้าทุก region เป็น clean_layout + orig==final = โค้ดที่แก้ (ซึ่งมักอยู่ใน bubble-fit path) ไม่ได้แตะ One-Punch เลย → regression = 0 แบบพิสูจน์ได้ ไม่ใช่แค่ดูภาพ.** (2026-07-01 height-fix+SIMPLE-A: One-Punch 7 region ทั้งหมด clean_layout orig==final → isolated ยืนยัน). ⚠️ matrix One-Punch อาจเผย defect ค้างของ One-Punch เอง (item 6 ぬ→LOOM ไม่แปล, item 8 clean_layout overflow) — คนละ class กับ fix ไทย แต่ต้อง list ตามกฎ (5). **🔴 แก้ heuristic ที่พิสูจน์แล้วว่าผิด/ไม่พอ (2026-07-02): "ทุก region clean_layout + orig==final → regression=0" = FALSE.** วัดจริงพบ One-Punch 6 region เป็น clean_layout orig==final **ทั้งหมด แต่ยัง oversized เกิน target** (fill_frac 1.2–2.45 = block ล้น avail; region ล่างขวา fill_frac 2.45 block_w 218 vs avail 87 → ล้นทับ art + clip). เพราะ `clean_layout_target_fs` (rendering/__init__.py:262) ดึง font ขึ้นตาม orig_fs (source JP 39-40px, item-2a "track orig_fs") + shrink loop (:286) เช็คแค่**ความสูง**ไม่เช็ค**กว้าง**. **orig==final แปลว่า "bubble-fit ไม่แตะ region นั้น" เท่านั้น — ไม่ได้แปลว่า "ตรง target"**; clean_layout เอง oversize ได้. **ต้องดูภาพเทียบ target ด้วยตา + เช็ค fill_frac (>1 = overflow) เสมอ ไม่ใช่เชื่อแค่ route/orig==final.** **+ config ต้อง faithful prod ครบทุก knob** — benchmark รอบแรก (2026-07-02) ตก `en_font=anime_ace_3.ttf` → worker fallback ไป Prompt (sans) → ดูเหมือน "font เพี้ยน" ทั้งที่ prod ถูก = false alarm. ก่อน benchmark ต้อง mirror Backend/.env ทุก MIT_* (en_font, uppercase, font_size_max, bubble_area_fit, clean_layout, ss4, det_bubble_seg ฯลฯ).

**กฎ meta (5) — บังคับ FULL MATRIX, ห้าม tunnel-vision (ผู้ใช้สั่ง 2026-07-01):** เวลา benchmark ทุกครั้ง **ต้องรายงานทุก item (1–12) ต่อทุกหน้า เป็นตาราง pass/fail + จุดที่เจอ** — แม้กำลังโฟกัสแก้ item เดียว ก็ห้ามรายงานแค่ item นั้นแล้วปัด item อื่นเป็น "แยกกัน/known/นอก scope". **เหตุผล: เคยพลาด (2026-07-01) — review 9 หน้าเพื่อ confirm item 9 แต่ดูแค่ "ตัดคำไหม" แล้วใส่ item 2/4 เป็น footnote บรรทัดเดียว ทำให้ user ต้องมาชี้ทีละจุดเอง ทั้งที่ item 2/4 เขียนใน MD นี้อยู่แล้ว.** การปัดว่า "known/separate" = ละเมิดกฎข้อ (1). ทุก defect ที่ตาเห็น (ไม่ว่า class ไหน, ไม่ว่าจะเป็นงานรอบนี้หรือไม่) ต้อง **list ออกมาให้ครบในรายงาน** ให้ user เห็นภาพรวมจริง ไม่ใช่ให้ user ไปหาเอง. รูปแบบ: ตาราง `หน้า × item(1-12)` ทุกช่องมี ✅/❌/⚪ + โน้ตจุด.

**Checklist (เทียบ original ทุกบับเบิล/ทุก caption):**
1. **text ว่าง/หาย** — บับเบิลที่ original มีคำแต่ฉบับแปลว่าง (มัก = บับเบิลซ้อน/ถูก patch อื่นบัง #436, หรือ region ถูก drop). เทียบจำนวนบับเบิลที่มีคำ original↔แปล
2. **ตัวเล็กกว่า original** — (a) display caption ที่ original ตัวใหญ่ (LOVE IS FORBIDDEN) แต่ render เล็ก = clean_layout flat font (แก้ #175: track orig_fs); (b) dialogue ในบับเบิลใหญ่ถูก route ไป clean_layout flat แทน bubble-fit (fills_bubble_width discriminator) — เทียบสัดส่วน font↔บับเบิล; (c) **บับเบิลแนวตั้งแคบ/ทรงสูง (narrow vertical/tall)** — original ตัวใหญ่เต็มความกว้างบับเบิล แต่ Thai render เล็กลอยกลาง เหลือที่ว่างบน-ล่างเยอะ (**user-annotated 2026-07-01: Gal Yome p20 "ต้องการถุงพลาสติกไหมครับ" + "1858 เยนครับ"** — original "Do you need a plastic bag?"/"It's 1858 yen." ตัวใหญ่เต็ม, Thai เล็กกว่าชัด). **⚠️ นี่คือ class ที่ user flag บ่อยสุด** — EN→TH dialogue ใน egg/oval/narrow bubble ใหญ่ render เล็ก ไม่เต็มบับเบิลแบบ original bold (Gal Yome p25 ล่าง, p26 ทั้ง 2, p12, p16 narration, p20 ทั้ง 2, p19 side box, p18). ต้องให้ Thai เต็มบับเบิลเท่า footprint อังกฤษเดิม — **เกณฑ์ตัดสิน: font height ของ Thai ต้อง ≈ original ในบับเบิลเดียวกัน, ไม่ใช่เล็กลงเพราะ wrap หลายบรรทัด**
3. **garbled/หลอน** — text มั่วที่ไม่ตรง original (มัก = det_sfx false-positive → VLM hallucinate, แก้ #278 ocr_read_real_text). **phantom "เงียบ"** = det_sfx false-positive ที่ read เป็น non-ASCII → หลุด gate #278 (ที่ drop เฉพาะ ASCII-readable) → แปลเป็น "เงียบ" แทรกใน dialogue (Gal Yome p09 ใน Notepad, p19)
4. **fade/จาง (text box fade)** — text จาง/หายบนพื้นมืด/**ขอบบับเบิล (lobe edge)** (เช็ค dark-bg + ขอบ heart/oval lobe โดยเฉพาะ — content-alpha เคยทำ fade ใน #266). **user-annotated 2026-07-01: Gal Yome p11 บับเบิลหัวใจล่างขวา "ถ้าพวกเธอเก็บความลับไว้ในบริษัท..." — ขอบขวา "เก็บความลับ/ไว้ในบริษัท" จางหายไปที่ขอบ lobe** (เดิม note แค่ "ภาย" — จริงๆ จางทั้งฝั่งขวา). **+ p25 บับเบิล tail เล็กบนขวา "พรุ่งนี้เรามีวันหยุด" (= "TOMORROW WE HAVE A REST DAY") fade หาย** (lobe เล็กซ้อนบับเบิลหลัก → text จาง/หลุดขอบ). เกณฑ์: glyph ต้องทึบสม่ำเสมอทั้งบับเบิล, ห้ามจางลงที่ขอบ patch/lobe; เทียบความเข้มขอบ↔กลางบับเบิลเดียวกัน. **หมายเหตุ: fade มักมาคู่กับ lobe เล็ก/tail bubble (text ใหญ่เกิน lobe → ดันออกขอบ → content-alpha จาง) — เช็ค item 4+7+2 พร้อมกันบน tail bubble**
5. **multi-lobe ไม่กระจาย** — 1 ประโยคที่ original แยกลงหลาย bubble lobe เชื่อมกัน (PRESIDENT!/WE ARE GOING TO GET MARRIED) แต่เรายัดรวม lobe เดียว lobe อื่นว่าง
6. **romaji/script ค้าง** — ชื่อ/SFX ไม่ถูก transliterate (TOUJOU FUYUKI, "파이" Korean leak, TAA-)
7. **overlap ซ้อนทับ** — SFX ซ้อน dialogue / 2 patch render ทับกัน garbled. instance: Gal Yome p11 บับเบิลหัวใจซ้อนแน่น 2 ใบ text ชนกัน (#436 กัน empty ได้แต่ยัง collide)
8. **clipped/ล้นกรอบ** — text ล้นออกนอกบับเบิล/โดนตัด (บับเบิลทรงแคบ/ถูกบัง)
9. **ตัดคำผิด/ขึ้นบรรทัดผิด (word-break)** — สำหรับภาษาที่ไม่มีช่องว่างระหว่างคำ (ไทย/ญี่ปุ่น/เขมร/ลาว) การ wrap ต้องตัดตาม **ขอบเขตคำ (dictionary/ICU segmentation)** — **ห้ามหักกลางคำ/กลาง cluster**. เช่น Gal Yome p25 "ข้างนอก"→"ข้า"/"งนอก", p18 "พยายาม"→"พยาย"/"ามให้" + "ไม่เป็น"→"ไม่เป็"/"น", p19 "ตลอด", เดรสถักหน้า "ความสั้น"/"เดรส"/"เครื่อง" หักกลางคำ. เทียบ: คำเดียวต้องไม่ถูกหักครึ่ง; พยัญชนะ+สระ/วรรณยุกต์ต้องอยู่บรรทัดเดียวกัน. **⚠️ class เด่นคู่กับ item 2** — EN→TH wrap ตัดกลางคำเพราะ wrapper ตัดตาม char/space ของอังกฤษ ไม่รู้จักขอบเขตคำไทย
10. **patch แตก pixel/low-res** — patch ที่ render (มัก = SFX) ขอบหยักแตกเป็น pixel/aliasing ต่างจาก dialogue ที่คมชัด = patch ถูก resample/upscale จาก low-res หรือ supersampling ไม่ทำกับ patch นั้น. instance: Gal Yome p11 SFX "ฮึย" แตก pixel. เทียบความคมของ glyph patch↔dialogue ในหน้าเดียวกัน
11. **inpaint ไม่สะอาด/ghost original text** — พื้นหลัง patch ลบ text เดิมไม่หมด เหลือเงา/เศษ original โผล่หลัง text แปล (ต่างจาก #436 overlap=2 patch ชน, อันนี้คือ inpaint mask ไม่ครอบ original เต็ม). instances: Gal Yome p19 มี original EN หลัง "ช่วงนี้เราไม่ค่อยได้คุยกันเท่าไหร่"; p27 บับเบิล cursive "What…here?" ไม่ถูกลบเลย. เช็คทุก patch ว่าพื้นหลังสะอาด ไม่มี ghost
12. **line-count บวม / UI strip ล้น (constrained horizontal element)** — กล่อง/แถบที่ original เป็น **1 บรรทัดแนวนอน** (title bar, ป้าย, แถบ UI, single-line caption) แต่ Thai แปลยาวกว่า → wrap เป็นหลายบรรทัด ล้นความสูงกล่อง/ทับขอบ UI. ต่างจาก item 2 (เล็กไป) และ item 8 (ล้นกรอบบับเบิล): อันนี้คือ **จำนวนบรรทัดไม่ match original ในกรอบที่ความสูงคงที่**. **user-annotated 2026-07-01: Gal Yome p09 title bar "Untitled - Notepad" (1 แถว) → "UNTITLED -NOTEPAD(ไม่มีชื่อ-บันทึกข้อความ)" กลายเป็น 3 แถว** ล้นแถบ title. เกณฑ์: ถ้า original = 1 บรรทัดในกรอบแนวนอนแคบ → ต้องลด font ให้คงจำนวนบรรทัด ≈ original (ยอม shrink ดีกว่า wrap ล้น), ไม่ใช่ wrap เพิ่มบรรทัด. เทียบจำนวนบรรทัด original↔แปล ในกรอบ UI/ป้าย/แถบ

**2 class เด่นสุด (pervasive เกือบทุกหน้า EN→TH):** item 2 (เล็กกว่า original) + item 9 (ตัดคำผิด). แก้ 2 อันนี้ก่อน impact สูงสุด.

**สถานะ + regression-watch pairing (2026-07-01):**
