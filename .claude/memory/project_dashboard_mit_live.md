The standalone **Dashboard** (`Dashboard/`, Next.js 16, port 4100, `bun dev` ‚Äî use **`localhost:4100`** not `127.0.0.1`, Next 16 blocks cross-origin HMR for raw IPs; public URL **`https://dashboard.hayateotsu.space`** via the cloudflared tunnel `1cc9b897-‚Ä¶`, ingress in `~/.cloudflared/config.yml` ‚Üí `localhost:4100`, `next.config.ts` `allowedDevOrigins` lists the host) connects to **MIT** live telemetry via OAuth.

> Tunnel/DNS note (2026-06-15): the `dashboard` CNAME lives in the **hayateotsu.space** zone. The cloudflared `cert.pem` was re-scoped from narze‚Üíhayateotsu to run `tunnel route dns` (old one backed up at `~/.cloudflared/cert.pem.narze.bak`; restore it if you ever need to `route dns` in narze.space). The running `tunnel run` uses the JSON credentials, not cert.pem, so re-scoping didn't disturb it. Built 2026-06-15 (PRD #279, ADR 016 / **ADR 017**). MIT-only slice (Backend/Frontend `/status` = #282/#283, akkanop-x refactoring ‚Äî see [[project-team-split]]).

**Auth = Frontend parity + GitHub, standalone (built 2026-06-15):** the Dashboard carries its OWN auth (does NOT depend on the Frontend) ‚Äî Email/password + Google + Facebook + **GitHub**, via the Frontend's **popup OAuth flow** (`signInWithOAuth({skipBrowserRedirect})` ‚Üí `app/auth/callback/page.tsx` postMessages the session back; avoids full-page redirect + surfaces errors in-app). In-app multi-provider **link/unlink** panel (`components/account-panel.tsx`; `lib/account.ts` + `lib/oauth.ts` pure-tested) so a dev links GitHub themselves. Role intent: Moderator‚ÜíGoogle, Admin‚Üíboth, Dev‚ÜíGitHub.

> **Manual-linking gotcha (the GitHub-login failure):** the project has manual account-linking on (the Frontend multi-platform feature). So a **fresh `signInWithOAuth({github})` onto an email that already has an account is REFUSED** by GoTrue ("Multiple accounts with the same email address in the same linking domain detected: default") ‚Äî anti-takeover, NOT a data bug (verified via Supabase MCP: 1 google user `9c7f7717-071a-40e1-9858-3458ea3251b5`, no dup). Fix: sign in with the existing provider (Google), then **link GitHub in the Account panel** (`linkIdentity`). After linking, `providers` includes github ‚Üí the dev gate passes.

**Flow:** browser ‚Üí Supabase OAuth/email (gate in `components/auth-gate.tsx`) ‚Üí session JWT ‚Üí `fetch /api/live` (same-origin, Authorization header) ‚Üí Next proxy `app/api/live/route.ts` forwards the JWT ‚Üí MIT `GET /status/stream` (SSE). MIT **verifies the token itself** via Supabase `GET /auth/v1/user` (`server/auth.py` ‚Äî **NOT PyJWT**; ADR 017 refines ADR 016's PyJWT assumption ‚Äî reason: Supabase now issues asymmetric `sb_publishable_‚Ä¶` keys that local HMAC can't verify; `getUser` is scheme-agnostic + matches the Backend's `verifyAccessToken`). No shared secret anywhere.

**Auth gotchas:**
- Login gate offers **GitHub (primary) + Google (secondary)**. **Dev-console access is forced onto GitHub** ‚Äî MIT enforces `MIT_DEV_REQUIRE_PROVIDER` (default `github`): a Google-only account is **denied dev** even if allowlisted (rationale: highest-privilege console = repo-collaborator GitHub identity; Google kept for future lower tiers). Set the env empty to disable during rollout.
- Staff gate = `staffLevel` claim (once the Supabase Custom-Access-Token hook lands) **OR** the dev's Supabase user-id in env **`MIT_STAFF_USER_IDS`** (comma-sep), **AND** the GitHub provider when enforced. **First sign-in ‚Üí 403** until the id is added; the dashboard then stays on mock with an "offline ¬∑ mock" badge. Add the id, restart MIT, ‚Üí live.
- **GitHub provider must be enabled in Supabase** (Authentication ‚Üí Providers ‚Üí GitHub + a GitHub OAuth App client id/secret, callback `https://<proj>.supabase.co/auth/v1/callback`) before the GitHub button works ‚Äî one-time setup the dev owns.
- MIT `/status*` env: `SUPABASE_URL`, `SUPABASE_ANON_KEY` (the public `sb_publishable_‚Ä¶`), `MIT_STAFF_USER_IDS`, `MIT_DEV_REQUIRE_PROVIDER` (github), `MIT_STATUS_INTERVAL_S` (3), `MIT_STATUS_REVALIDATE_S` (60), `MIT_DIAG_INTERVAL_S` (30, throttles the gateway probe).
- Dashboard env (`.env.local`): `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `MIT_STATUS_URL` (server-side, where `/api/live` forwards ‚Äî default `http://127.0.0.1:5003`).

**Launch/test:** For E2E without the GPU worker, run MIT **parent-only** on a spare port (`server/main.py --host 127.0.0.1 --port 5013`, no `--start-instance`) ‚Äî real GPU metrics still flow via `nvidia-smi` (don't need a worker), `/status` reports `workers:0` ‚Üí status `down` but real host/GPU. Point the dev server at it: `MIT_STATUS_URL=http://127.0.0.1:5013 bun dev -p 4100`. Reject paths (no-token/garbage ‚Üí 401) are curl-testable against real Supabase; the **valid-token 200 needs a human Google sign-in** (OAuth can't be driven headlessly). `getSession()` can stall headless (navigator.locks) ‚Üí the gate has a 2.5s splash timeout so it never hangs. See [[project_mit_launch_env]] for the full GPU restart recipe.

> **Restart-env gotcha (2026-06-16):** `server/main.py` does **NOT** `load_dotenv` ‚Äî so `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `MIT_STAFF_USER_IDS`, `MIT_DEV_REQUIRE_PROVIDER`, `MIT_DIAG_*` must be **exported in the launch shell**, they are NOT in `MIT/.env` (which only holds `CUSTOM_OPENAI_*` etc.). Reconstruct on restart: Supabase URL/key ‚Üê `Dashboard/.env.local` (`NEXT_PUBLIC_SUPABASE_URL`/`_ANON_KEY`); the gateway probe falls back to `MIT/.env`'s `CUSTOM_OPENAI_API_BASE`/`_KEY`/`_MODEL` (read as `MIT_DIAG_BASE_URL`/`_API_KEY`/`_MODEL`). The working dev staff id = `9c7f7717-071a-40e1-9858-3458ea3251b5`. Force **parent-only** (no `--start-instance`) when free RAM < 15 GB ([[project_dev_commit_memory]]) ‚Äî the Qwen3 load OOMs otherwise; parent-only still serves `/status*` immediately (no `/ready` wait, no model). `Start-Process .venv\Scripts\python.exe -u server/main.py --host 127.0.0.1 --port 5013` after `$env:`-setting the above.

**Modules:** MIT pure (import-light, TDD): `status_snapshot` / `auth` / `status_hub` (asyncio pub/sub, event push) / `status_stream` (SSE hybrid: sample metrics + push events). Dashboard pure: `lib/live.ts` (`parseSseFrames`) / `lib/live-map.ts` (`mapMitSnapshot`); folds via the committed `lib/snapshot.ts` reducer.
