02 Why it's hard — two complementary render campaigns

Not "one branch has render." main and landing each built a different half of the render stack on orthogonal axes. Neither dominates — so the merge is a feature-merge at function granularity, not a branch pick. The crux is one function: resize_regions_to_font_size() in rendering/__init__.py — take landing → lose KP/#178/#183; take main → lose display-SFX/cap/dedup. Both must survive.

05 The convergence plan — where to freeze, merge, rebase, deploy

A

Freeze & clean DEV GATE FREEZE

Commit the 320-file perf WIP to wip/perf-freeze, restore a clean perf. Triage: 25 discard (memory already migrated) · 264 trash (screenshots/logs) · 13 render modules kept for reference · 11 tool scripts. The one true blocker — only the dev knows what to keep.

B

Integration branch AGENT MERGE

integrate/render-reconcile off main. Cherry-pick perf's unique winners first (#623 thinking-fix, Stage A), then merge landing --no-squash (~23 conflicts surface here).

C

Feature-merge the render set AGENT · TDD

Resolve in order: ① config.py + mit-config.ts together (flag contract) → ② hand-merge resize_regions_to_font_size() (keep main spine + landing SFX/dedup) → ③ inpaint/rescue wiring unit → ④ take landing wholesale for dead-on-main helpers + land the unique #182 vertical slice. Also: renumber the duplicate ADR 023.

D

Quality gates — both BLOCKING AGENT · GPU

Regenerate golden .npz from reconciled code. Render gate: bench vs landing baseline (One-Punch + 8-pt checklist). Translation gate (dev hard-constraint): text equal-or-better than baseline — no dropped/garbled bubbles, index-aligned, no romaji; includes the #623 thinking-off vs thinking-on A/B. Commit PNG + text-diff + report.

E

Promote & deploy DEV GATE REBASE-SAFE

Merge integrate → main (classifier blocks agent self-merge to default). Fast-forward main → perf. Bump MIT_RENDER_VERSION, enforce lama_large + selective_flux=off + KP gated by render.knuth_plass. Keep a rollback branch.
