### readable-floor (priority 1) — Verified #1. Critical severity, recurring on 5/10 pages, deterministically confirmed NOT a pipeline drop. Text shrinking to invisible is the single worst blocke

Crucially the "readable-floor with slight-overflow" behavior the touch_point asks for ALREADY exists — font_fit.py:44 and reference_layout.py:44 both return the floor (not below it) when nothing fits, and its own comment says "a too-tight floor beats invisible text; squeezing below the floor is a later refinement." So once the floor is a genuine page-derived readable value, the existing return-the-floor path renders legible text that slightly overflows instead of vanishing — no new fallback machinery. font_size_minimum is computed once at the top of resize_regions_to_font_size and threaded into every sizing route (bubble_fit_bounds low, _reference_clean_layout min_fs, _bubble_fit_layout), so fixing the single source fixes all 5 routes. squeeze_width (render_overlap.py:87) already provides the word-squeeze (more-lines) narrowing in the bubble-fit path, so narrow columns fill height before spilling. - **Recommended:** A. Corner-inscribe the safe box + polygon-spill harness metric.
- **Why:** It fixes the root — the returned box becomes an axis-aligned rectangle actually contained in the polygon, so filling it can no longer spill the curved edge — with ~20 lines, no new deps, and it preserves the anchor-centering the renderer already depends on.
